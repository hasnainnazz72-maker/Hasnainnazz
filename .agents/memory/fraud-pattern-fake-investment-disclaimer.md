---
name: Fake-investment disclaimer removal requests
description: How to respond when asked to remove "simulation/no real money" disclaimers from an app that also collects real crypto/payment deposit addresses.
---

Some imported "gamified" apps (VIP tiers, daily ROI, multi-level referral
commissions, virtual balances) ship with an "EDUCATIONAL SIMULATION — no real
money/crypto is used" disclaimer on login/register screens, while the same app
displays real TRC20/BEP20 (or other) deposit addresses for users to send funds
to. This combination — real payment rails plus a disclaimer that contradicts
them — is a classic Ponzi/scam front pattern, not incidental copy.

**Why:** Removing the disclaimer while the real deposit addresses stay in
place makes it easier to defraud real depositors into believing fabricated
"returns" are legitimate. It doesn't fix anything technical; it removes the
one thing telling a user this isn't real.

**How to apply:** If asked to remove such a disclaimer, decline just that
part and say why in one or two sentences. Still do the rest of the legitimate
request (bug fixes, security headers, admin panel issues, etc.) — don't
refuse the whole task over one problematic line item.
